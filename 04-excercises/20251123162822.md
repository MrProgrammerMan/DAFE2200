# Ukesoppgaver - uke 10
1. De 7 prinsippenes påvirkning av testing gjennom produktets livsløp
    1. Vi er nødt til å være realistiske om hva vi kan teste. Ettersom produktet blir mer komplekst vil dette være mer relevant. Vi må akseptere at feil kan oppstå og sette av tid til å håndtere dette.
    2. Vi kan rett og slett ikke teste alt, men må heller fokusere testressursene våre nøye der de har mest effekt. Dette vil også være viktig ettersom vi vokser som produkt. Testing kan ta ekstremt lang tid etter hvert hvis vi prøver å teste alt.
    3. Dersom vi ikke tester tidlig kan vi risikere å behøve store endringer i produktet. Tester vi tidlig unngår vi ressurssløsing.
    4. Ettersom programmet blir større blir det mer relevant å undersøke koden rundt der feil oppstår. Dette gjelder også ved feil etter deployment i vedlikeholdsfasen.
    5. Dersom vi tester kun etter én type feil kan det oppstå andre feil. Igjen påvirker kompleksitet dette. Vi må også være påpasselige når vi fikser feil.
    6. Vi er nødt til å teste i forskjellige miljøer, og holde testingen proposjonal til behovet.
    7. Fravær av feil betyr ikke korrekt.
2. Pilene betyr mulighet til å forberede tester basert på stedet man er i V-en. Når man samler krav kan man forberede akseptansetester osv. Systemtesting handler mer om helhetsting som brukeren ikke ser.
3. Statisk vs dynamisk
    1. Statisk testing kjører ikke koden. Inkluderer statisk analyse-verktøy, dead code detection, linting, etc.
    2. Dynamisk testing kjører koden. Enhets-, integrasjons-, system-tester.
4.
    1.  Test-typer er statiske, dynamiske.
    2.  Test-nivåer er enhets, integrasjons, system
    3.  Test-teknikker er spesifakasjonsbaserte og derunder, black box, white box, strukturbaserte, erfaringsbaserte, gransking, statisk analyse
5.  Testteknikker systematiserer testing og gjør det mulig å skrive rapporter om testing som kan etterprøves.
    1.  Vi har veldig mange test-teknikker. Blackbox vil si at vi ikke kjenner koden. White box vil si at vi kjenner koden.
